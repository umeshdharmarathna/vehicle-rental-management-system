# Vehicle Management System

A simple, console-based Java application designed to manage vehicle rentals. This project demonstrates core Object-Oriented Programming (OOP) concepts such as Inheritance, Polymorphism, Abstraction, and Encapsulation.

## 🚀 Features

The system provides an interactive menu with the following capabilities:
* **Add Vehicles**: Add different types of vehicles to the system (Cars, Bikes, Vans) with specific attributes.
* **View All Vehicles**: Display a list of all vehicles currently in the system along with their details.
* **Rent a Vehicle**: Rent an available vehicle and calculate the total rental cost based on the number of days and vehicle-specific rates.
* **Return a Vehicle**: Mark a rented vehicle as available again.
* **Search & Filter**: Search for a specific vehicle by its unique ID or filter the list of vehicles by their type.
* **Remove Vehicles**: Delete a vehicle from the system (only if it is not currently rented).
* **Track Income**: Keep track of the total rental income generated by the system.
* **Data Persistence**: Automatically loads existing data on startup and saves new data to a `vehicles.txt` file before exiting.

## 💻 Technical Details & OOP Concepts

This application is built entirely in **Java** and utilizes several key OOP principles:
* **Abstraction**: The `Vehicle` class is abstract, defining a blueprint and abstract methods like `calculateRentalCost()` and `toSaveData()` that must be implemented by subclasses.
* **Inheritance**: `Car`, `Bike`, and `Van` classes extend the base `Vehicle` class, inheriting common properties (ID, brand, model, base rate) while introducing their own specific attributes (number of seats, engine CC, cargo capacity).
* **Polymorphism**: The system handles different vehicle types uniformly through the `Vehicle` reference type, overriding methods to provide specific implementations for cost calculation and data formatting.
* **File Handling**: Uses `java.io.File`, `java.util.Scanner`, and `java.io.PrintWriter` to ensure data remains persistent between application restarts.
